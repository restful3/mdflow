# M2b — Marker(GPU) PDF 컨버터 구현 계획

**작성일**: 2026-05-23
**선행**: M2a `v0.2.0-m2a` (GPU 라우팅 배관: `gpu_lock` + `queued` 이벤트, disconnect-safe semaphore release). M5 `v0.6.0-m5` (현재 마스터).
**범위 기준**: `docs/specs/2026-05-22-m2-pdf-design.md` §1.2 / §10 "M2b drop-in".
**호스트**: 이 세션의 호스트(RTX 3060 12GB, CUDA driver 12.6) — 실 GPU 검증 가능.

---

## 1. 목표

M2 spec이 만든 GPU 라우팅 배관(`convert.py`의 `if lr.converter.requires_gpu:` 분기 + `gpu_semaphore(1)` 직렬화)에 **Marker(고품질 GPU 경로) 컨버터**를 등록한다. registry의 **first-wins + `can_handle` 게이팅**으로 자동 분기:

- GPU 있고 `marker-pdf`/`torch` 설치돼 있고 `MDFLOW_FORCE_CPU=0`이면 → Marker
- 그 외엔 `can_handle=False` → PyMuPDF(M2a 등록 그대로)로 떨어짐

별도 fallback chain 실행기/런타임 에러 폴백 **없음** (§6 예외-삼킴 회피 + YAGNI, M2 spec §5와 동일).

## 3. MarkerConverter 사양

```python
# src/mdflow/converters/marker.py
class MarkerConverter:
    name = "pdf-marker"
    formats: tuple[str, ...] = ("pdf",)
    requires_gpu = True

    def can_handle(self, ctx: ConversionContext) -> bool:
        # 1) format match
        if ctx.format not in self.formats:
            return False
        # 2) MDFLOW_FORCE_CPU 우회
        if _force_cpu():
            return False
        # 3) torch + CUDA 가용성
        if not _cuda_available():
            return False
        # 4) marker import 가능
        return _marker_available()

    def convert(self, ctx, progress):
        progress("load", 5)
        models = _load_models()                # marker.models.create_model_dict()
        try:
            progress("parse", 20)
            # marker는 path 입력 — bytes를 임시 파일로 저장
            with tempfile.NamedTemporaryFile(suffix=".pdf", delete=False) as f:
                f.write(ctx.data)
                tmp = Path(f.name)
            try:
                progress("render", 50)
                rendered = _marker_convert(tmp, models)   # MarkerPdfConverter(artifact_dict=models)(path)
                text, _, _images = _text_from_rendered(rendered)
            finally:
                tmp.unlink(missing_ok=True)
            progress("done", 100)
            return ConversionResult(
                markdown=text.strip(),
                metadata={"engine": "marker", "pages": rendered.metadata.get("page_stats", [{}]).__len__()},
            )
        finally:
            # PaperFlow VRAM 정리 패턴 (CLAUDE.md §1.4 참고)
            del models
            gc.collect()
            try:
                import torch
                if torch.cuda.is_available():
                    torch.cuda.empty_cache()
            except Exception:
                pass
```

**핵심 결정**:

- **모델 로드는 매 호출** — Marker 단일 모델 인스턴스 캐시는 v2(현재 직렬화 = 세마포어(1), VRAM 안전이 1순위). 로딩 비용은 PaperFlow가 합의했듯 첫 호출에 ~수 초. 추후 캐싱은 별도 슬라이스.
- **bytes → 임시 파일** — Marker 공식 API가 path 기반. M3a의 `TemporaryDirectory` 패턴 재사용.
- **이미지 드롭** — `text_from_rendered`의 images는 무시 (M1b/M3b와 동일 정책).
- **`_text_from_rendered`/`_marker_convert`/`_load_models` lazy import** — 모듈 import 시점에 marker를 끌어오지 않는다(HWP `[hwp]` extra 격리와 동일 패턴).
- **`del model; gc.collect(); torch.cuda.empty_cache()`** — PaperFlow 누수 회피 패턴(CLAUDE.md §1.4)을 `finally`에 강제.
- **예외 자체 catch 없음** (§6) — Marker/torch 예외 전파 → `run_conversion`이 `CONVERSION_FAILED` wrap → SSE `error`. VRAM 정리는 `finally`라 예외 경로에서도 실행.

## 4. 테스트 전략

### 4.1 단위 (CI에서 항상 실행)

`tests/converters/test_marker.py`:

1. `requires_gpu == True` / `formats == ("pdf",)` / `name == "pdf-marker"`
2. `can_handle` 게이팅 표:
   - `MDFLOW_FORCE_CPU=1` → False (monkeypatch env)
   - `_cuda_available` → False (monkeypatch) → False
   - `_marker_available` → False (monkeypatch) → False
   - 모두 True → True
3. `convert()` monkeypatch — `_load_models`/`_marker_convert`/`_text_from_rendered`를 stub로 대체 → 결정적 markdown 반환 + progress 호출 + VRAM cleanup 호출 검증

## 6. 리스크 / 노트

- **cu124 wheel size** — torch CUDA ~2.5GB. 첫 설치만. requirements 캐시.
- **Marker 모델 다운로드** — 첫 inference ~2GB(전역 HF 캐시). `pytest -m gpu` 첫 호출에 분 단위. 로컬 캐시되면 이후 빠름.
- **VRAM** — Marker base 모델 ~5\~8GB. RTX 3060 12GB로 여유. semaphore(1)가 동시 1개 보장.
- **Marker API 안정성** — `marker-pdf` 2.x API. 설치 후 정확 import 경로 확정 (`marker.converters.pdf.PdfConverter` vs `marker.convert.convert_single_pdf`). Task 0 직후 확인.
- **MCP 미적용** — M2b는 SSE 경로 `requires_gpu` 직렬화에만 합류. MCP transport는 별도 runtime이라 `gpu_semaphore` 게이팅 없음(spec §10 노트 그대로). MCP GPU 직렬화는 별도 슬라이스(필요 시).
- **CLI 미적용** — CLI(`mdflow convert`)는 동기 1회 변환이고 `requires_gpu` 분기를 거치지 않음(`ConversionService.convert` 직접 호출). MCP와 동일하게 별도 슬라이스 필요.
